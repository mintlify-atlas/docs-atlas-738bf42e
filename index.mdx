Patrimonio Gateway is the central API Gateway for the Atotonilco de Tula municipal heritage asset management system. It sits in front of all backend microservices — handling authentication, intelligent routing, CORS, and automatic service wake-up — so your frontend or integration only needs one base URL.

## How it works

The Gateway exposes a single base URL and routes each request path to the appropriate upstream microservice. All requests pass through CORS handling and request logging before reaching their destination.

<Steps>
  <Step title="Send a request to the Gateway">
    Use the Gateway's base URL (e.g., `https://your-gateway.onrender.com`) instead of calling microservices directly.
  </Step>
  <Step title="Authenticate with a Bearer token">
    Obtain a JWT from `POST /auth/login`, then include it as `Authorization: Bearer <token>` on all subsequent requests.
  </Step>
  <Step title="The Gateway routes your request">
    Based on the path prefix (`/auth`, `/bienes`, `/roles`, `/usuarios`, `/api/upload`, `/importador`), the Gateway proxies your request to the correct microservice.
  </Step>
  <Step title="Receive the response">
    The microservice response is returned directly to you. If the service was asleep (Render.com cold start), the Gateway wakes it automatically before forwarding.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Intelligent wake-up" icon="bed-pulse">
    Automatically pings sleeping Render.com services before forwarding requests, preventing cold-start failures
  </Card>
  <Card title="Auth header forwarding" icon="shield-halved">
    Transparently passes `Authorization: Bearer` tokens to all upstream services
  </Card>
  <Card title="Proxy error recovery" icon="arrows-rotate">
    On proxy failure, retries non-GET requests after waking the target service
  </Card>
  <Card title="CORS with credentials" icon="globe">
    Configured for cross-origin requests with credentials support for browser clients
  </Card>
</CardGroup>
