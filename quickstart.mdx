---
title: "Quickstart: Make your first Gateway request in minutes"
description: "Start the Patrimonio Gateway locally, verify it is running, obtain a JWT from the auth service, and make your first authenticated request to the bienes service."
---

This guide walks you through starting the Gateway, confirming it is healthy, logging in to get a JWT, and then calling a protected endpoint. All examples use the default local address `http://localhost:3000`, but the same steps apply to any deployed instance — just swap the base URL.

<Steps>

<Step title="Start the Gateway">

Clone the repository and install dependencies if you have not already:

```bash
git clone https://github.com/Dari175/Patrimonio-Gateway.git
cd Patrimonio-Gateway
npm install
```

Then start the server:

```bash
node index.js
```

You should see the following output in your terminal:

```
Gateway corriendo en puerto 3000
```

To use a different port, set the `PORT` environment variable before starting:

```bash
PORT=8080 node index.js
```

<Tip>
  Add a `"start": "node index.js"` script to your `package.json` so you can run `npm start` instead.
</Tip>

</Step>

<Step title="Check the health endpoint">

Confirm the Gateway is running by hitting the `/health` endpoint:

<CodeGroup>

```bash curl
curl http://localhost:3000/health
```

```javascript JavaScript fetch
const res = await fetch('http://localhost:3000/health');
const data = await res.json();
console.log(data); // { ok: true }
```

</CodeGroup>

A healthy Gateway responds with:

```json
{ "ok": true }
```

<Note>
  The `/health` endpoint does **not** ping the upstream microservices — it only confirms the Gateway process itself is responding. Use it as a liveness probe in your deployment platform.
</Note>

</Step>

<Step title="Log in to get a JWT">

Send your credentials to `POST /auth/login`. The Gateway handles this route manually: it wakes the auth service if needed, then forwards your request to `POST /login` on `patrimonio-apiservice-auth.onrender.com`.

<CodeGroup>

```bash curl
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "your-password"}'
```

```javascript JavaScript fetch
const res = await fetch('http://localhost:3000/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'admin', password: 'your-password' })
});

const { token } = await res.json();
console.log(token); // eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

</CodeGroup>

A successful login returns a JSON object containing your JWT:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<Warning>
  The first login request may take several seconds if the auth service is cold. The Gateway automatically attempts to wake it — this is expected behavior on the free Render.com tier.
</Warning>

</Step>

<Step title="Make an authenticated request">

Use the token from the previous step to call a protected endpoint. The example below fetches the list of heritage assets from the `/bienes` route:

<CodeGroup>

```bash curl
curl http://localhost:3000/bienes \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```javascript JavaScript fetch
const res = await fetch('http://localhost:3000/bienes', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const bienes = await res.json();
console.log(bienes);
```

</CodeGroup>

The Gateway proxies this request to `https://bienes-service-nldc.onrender.com/api/bienes`, forwarding your `Authorization` header automatically.

<Tip>
  All routes except `/health` require a valid Bearer token. Store your token in a variable or environment and attach it to every request.
</Tip>

</Step>

</Steps>
