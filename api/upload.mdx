The Upload routes let you attach images to heritage asset records in the patrimony system. The Gateway forwards all `/api/upload/*` requests to the `patrimonio-loadimages` microservice, preserving the full `/api/upload/` path prefix on the upstream service. File transfers use `multipart/form-data` encoding, which the Gateway passes through without modification.

## How routing works

Requests to `/api/upload/*` are forwarded to `https://patrimonio-loadimages.onrender.com`. The path is rewritten so that the `/api/upload` prefix is **re-applied** on the upstream side, keeping the upstream service's own routing intact.

| Gateway path | Upstream path |
|---|---|
| `POST /api/upload/foto` | `POST /api/upload/foto` |
| `POST /api/upload/documento` | `POST /api/upload/documento` |

The Gateway also wakes the upload service before forwarding if the service is sleeping.

## Authentication

Every request must include a Bearer token in the `Authorization` header.

```http
Authorization: Bearer <your-token>
```

## Request headers set by the Gateway

| Header | Value | Description |
|---|---|---|
| `Authorization` | Forwarded from client | Bearer token passed through as-is |
| `ngrok-skip-browser-warning` | `true` | Suppresses ngrok interstitial pages |
| `x-module` | `roles` | Set automatically by the Gateway on every upload request |

<Note>
  The Gateway sets `x-module: roles` on all requests to this route. This is the value defined in the source code (`index.js:256`) and is not configurable per-request.
</Note>

## Proxy settings

| Setting | Value |
|---|---|
| Target | `https://patrimonio-loadimages.onrender.com` |
| `changeOrigin` | `true` |
| `proxyTimeout` | 20 000 ms |
| `timeout` | 20 000 ms |
| Error recovery | Yes тАФ uses `createSafeProxy` with retry on non-GET errors |

## Uploading a file

Use `multipart/form-data` encoding when sending files. Do **not** set `Content-Type` manually when using `curl -F` or the browser `FormData` API тАФ the boundary value is set automatically.

<Note>
  Always use `multipart/form-data` for file uploads. Sending `application/json` or `application/octet-stream` will not be accepted by the upstream image service.
</Note>

<CodeGroup>

```bash curl
curl -X POST https://<gateway-host>/api/upload/foto \
  -H "Authorization: Bearer <your-token>" \
  -F "file=@/path/to/image.jpg"
```

```javascript fetch
const formData = new FormData();
formData.append('file', fileInput.files[0]);

const response = await fetch('https://<gateway-host>/api/upload/foto', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <your-token>',
    // Do NOT set Content-Type here тАФ the browser sets it with the boundary
  },
  body: formData,
});

const result = await response.json();
```

</CodeGroup>

## Error responses

| Status | Meaning |
|---|---|
| `401` / `403` | Invalid or missing Bearer token (returned by the upstream service) |
| `502` | Upstream service unreachable or timed out тАФ non-GET requests are retried once by the safe proxy |
