The Importador routes handle bulk data entry for the patrimony system. Instead of creating asset records one by one, you can upload an Excel file containing multiple asset rows and the Gateway will forward it to the `patrimonio-importexeldb` microservice, which parses and persists the records to the database. This is the recommended path for initial data loads and batch updates.

## How routing works

Requests to `/importador/*` are forwarded to `https://patrimonio-importexeldb.onrender.com/importar`. The Gateway strips the `/importador` prefix from the path and appends the remainder to the target URL — which already contains `/importar`.

| Gateway path | Upstream URL |
|---|---|
| `POST /importador` | `POST https://patrimonio-importexeldb.onrender.com/importar` |
| `POST /importador/lote` | `POST https://patrimonio-importexeldb.onrender.com/importar/lote` |

<Note>
  The upstream target is `https://patrimonio-importexeldb.onrender.com/importar` — the `/importar` segment is part of the target base URL defined in the Gateway configuration (`index.js:305`), not part of the rewritten path. You do not need to include `/importar` in your request path.
</Note>

The Gateway wakes the importador service before forwarding the request if the service is sleeping.

## Authentication

Every request must include a Bearer token in the `Authorization` header.

```http
Authorization: Bearer <your-token>
```

## Request headers set by the Gateway

| Header | Value | Description |
|---|---|---|
| `Authorization` | Forwarded from client | Bearer token passed through as-is |
| `ngrok-skip-browser-warning` | `true` | Suppresses ngrok interstitial pages |
| `x-module` | `importador` | Set automatically by the Gateway on every import request |

## Proxy settings

| Setting | Value |
|---|---|
| Target | `https://patrimonio-importexeldb.onrender.com/importar` |
| `changeOrigin` | `true` |
| `proxyTimeout` | 20 000 ms |
| `timeout` | 20 000 ms |
| Error recovery | Yes — uses `createSafeProxy` with retry on non-GET errors |

## Importing an Excel file

Send a `multipart/form-data` request with your Excel file attached. The upstream service reads the spreadsheet and inserts each row as a patrimony asset record.

<Tip>
  Use `.xlsx` format (Excel 2007 and later) for your import file. Ensure column headers match the field names expected by the upstream service. Sending a `.csv` or `.xls` file may result in a parse error from the importador service.
</Tip>

<Steps>
  <Step title="Prepare your Excel file">
    Create an `.xlsx` spreadsheet with one row per asset. Include the required columns such as `nombre`, `descripcion`, and `valor`. Save the file locally.
  </Step>
  <Step title="Send the import request">
    Upload the file using `multipart/form-data`. Do not set `Content-Type` manually — let the HTTP client set it with the correct multipart boundary.

    <CodeGroup>

    ```bash curl
    curl -X POST https://<gateway-host>/importador \
      -H "Authorization: Bearer <your-token>" \
      -F "file=@/path/to/bienes.xlsx"
    ```

    ```javascript fetch
    const formData = new FormData();
    formData.append('file', fileInput.files[0]);

    const response = await fetch('https://<gateway-host>/importador', {
      method: 'POST',
      headers: {
        Authorization: 'Bearer <your-token>',
        // Do NOT set Content-Type — the browser sets it with the boundary
      },
      body: formData,
    });

    const result = await response.json();
    ```

    </CodeGroup>
  </Step>
  <Step title="Check the response">
    A successful import returns a `200` or `201` response with a summary of records created. Review any error messages in the response body for rows that could not be processed.
  </Step>
</Steps>

## Error responses

| Status | Meaning |
|---|---|
| `401` / `403` | Invalid or missing Bearer token (returned by the upstream service) |
| `422` | Malformed spreadsheet or missing required columns (upstream validation error) |
| `502` | Upstream service unreachable or timed out — non-GET requests are retried once by the safe proxy |
